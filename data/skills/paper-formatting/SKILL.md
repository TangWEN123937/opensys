## 论文排版定稿流程（本技能被激活时执行）

### 第一步：结构完整性检查

确认论文包含以下完整组成部分（根据论文类型增减）：

**期刊论文**：
- [ ] 标题（中英文）
- [ ] 作者信息（姓名、单位、通讯方式）
- [ ] 摘要（中英文）
- [ ] 关键词（中英文）
- [ ] 正文各章节
- [ ] 参考文献
- [ ] 基金项目（如有）

**学位论文**：
- [ ] 封面（题目、作者、导师、学校、日期）
- [ ] 声明页（原创性声明、授权声明）
- [ ] 中文摘要 + 关键词
- [ ] 英文摘要 + Keywords
- [ ] 目录
- [ ] 图表目录（如有大量图表）
- [ ] 正文各章节
- [ ] 参考文献
- [ ] 致谢
- [ ] 附录（如有）
- [ ] 作者简介/在学期间发表论文（如有要求）

### 第二步：格式规范化

#### 通用学术排版规范（Markdown 输出）

1. **标题层级**：
   - 一级标题：`# 第一章 绪论`（章标题）
   - 二级标题：`## 1.1 研究背景`（节标题）
   - 三级标题：`### 1.1.1 ...`（小节标题）
   - 最多使用三级标题

2. **段落格式**：
   - 段落之间空一行
   - 首行无需缩进（Markdown 特性）
   - 正文使用统一的表述风格

3. **图表规范**：
   - 图标题在图下方：`图 X-Y 图标题`
   - 表标题在表上方：`表 X-Y 表标题`
   - 图表编号按章编号（如第三章第二个图为"图 3-2"）
   - 图表必须在正文中被引用（"如图 3-2 所示"）
   - 数据来源标注在图表下方

4. **公式规范**（如有）：
   - 使用 LaTeX 语法：`$$公式$$`
   - 公式编号右对齐：`$$E = mc^2 \tag{3-1}$$`
   - 正文引用"如式 (3-1) 所示"

5. **引用格式核对**：
   - 确认引用标记格式全文统一
   - 参考文献列表格式与 Phase 7 产出一致

### 第三步：终稿元信息

在论文开头添加元信息（供人工排版参考）：

```markdown
---
title: [论文标题]
title_en: [English Title]
author: [作者]
advisor: [导师]（学位论文）
institution: [学校/单位]
date: [日期]
word_count: [约 N 字]
paper_type: [期刊/学位/会议]
citation_format: [GB/T 7714 / APA]
---
```

### 第四步：通读校对与输出

1. **通读全文**：
   - 检查错别字、语病
   - 检查前后表述一致性（同一概念全文统一用词）
   - 检查数字和数据准确性
   - 检查引用编号连续性

2. **输出终稿文件**：
   - 使用 `write_and_run_script` 将终稿写入文件
   - 终稿写入 `<任务目录>/output/[主题简称]_论文终稿.md`
   - 如用户提供了 Word 模板，可考虑使用 python-docx 生成 .docx 文件

3. **文件整理**：
   终稿生成后，将任务过程中产生的所有中间文件（分章节草稿、摘要草稿、引用映射等）统一归档到 `drafts/` 子目录，`output/` 目录只保留最终交付物：
   ```bash
   cd <任务目录>
   # 将 output/ 中的过程文件移入 drafts/（保留终稿和参考文献）
   mv -f output/chapter_*.md output/abstract.md output/paper_draft.md output/citation_map.json output/outline*.md output/section_*.md drafts/ 2>/dev/null
   # 将任务根目录下的散落中间文件也移入 drafts/
   mv -f *.tmp *.bak drafts/ 2>/dev/null
   echo "✅ 过程文件已归档到 drafts/"
   echo "--- output/ 最终交付物 ---"
   ls -la output/
   echo "--- drafts/ 过程文件 ---"
   ls -la drafts/
   ```
   最终目录结构应为：
   ```
   <任务目录>/
     ├── output/      # 仅终稿交付物
     │   ├── [主题]_论文终稿.md   — 完整终稿（用户交付物）
     │   ├── references.md        — 参考文献列表
     │   └── reference_report.md  — 引用核对报告
     ├── drafts/      # 所有过程文件（章节草稿、摘要草稿、引用映射等）
     └── downloads/   # 浏览器下载的文件（PDF 等）
   ```

### 注意事项

- **保持原意**：排版阶段不改变原文内容和论点
- **格式优先参考**：如用户提供了学校/期刊模板，以模板要求为准
- **中英文混排**：中文论文中的英文专有名词保持原文，中英文之间加空格
- **Markdown 局限性**：Markdown 无法完全实现学术排版（如页眉页脚、双栏等），需在元信息中标注供用户后续在 Word/LaTeX 中手动调整
- **脚本鲁棒性**：使用 `write_and_run_script` 处理论文文件时，不要硬编码假设特定 Markdown 格式（如 `## 摘要`），应先读取文件探测实际格式再处理。对 `.split()` / `.index()` 等操作必须加异常处理，避免 `IndexError` 导致脚本中断
- **过程文件必须归档**：终稿输出后必须执行文件整理，`output/` 中不允许残留草稿和中间产物
