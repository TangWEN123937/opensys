## ⛔ 强制规则

### 导航与登录

| 编号 | 规则 | 必须 | 禁止 |
|------|------|------|------|
| R1 | 直达链接 | 使用直达 URL 导航 | 从首页逐步点击菜单 |
| R2 | 登录协助 | 调用 `ask_user_for_browser` 请求用户扫码/输密码 | 尝试自动登录 |
| R3 | 安全验证 | 遇到验证码调用 `ask_user_for_browser` | 自行处理验证码 |
| R4 | 页面等待 | 页面加载后等待 2-3 秒再操作 | 页面未渲染完就操作 |

### 表单填写（核心，防止内容丢失）

| 编号 | 规则 | 必须 | 禁止 |
|------|------|------|------|
| R5 | 输入方式 | 用 `input` action 或 `send_keys` 填写所有字段 | 用 `evaluate`（JS）修改表单值（框架不感知，内容会丢失） |
| R6 | 两步填写 | 先单独填收件人并确认标签（Step 2a），再填主题+正文（Step 2b） | 收件人未确认就填其他字段（自动补全会抢焦点串内容） |
| R7 | 收件人格式 | 填写完整邮箱地址（如 `xxx@qq.com`） | 填写中文姓名或昵称 |
| R8 | 收件人确认 | 输入后按 **Enter** 确认 → 等待 **1 秒** | 输入后不按 Enter 就填其他字段（自动补全未完成，内容丢失） |
| R9 | 自动补全标签 | 收件人显示为数字标签（如 `1720778106`）是**正常行为**，直接继续 | 看到不含 `@` 就反复清除重新输入（会陷入死循环） |
| R10 | 主题填写 | 点击主题字段 → **优先用 `send_keys` 直接打字**（shadow DOM 元素，`input` action 常失败）→ **等待 1 秒** → **点击正文区域** | 用 `input` action + `clear: True`（shadow DOM 会触发字段重置，内容丢失）；用 Tab/Enter 确认主题；输入后不等待就点击其他区域；反复重试同一方法超过 2 次 |

### 发送与验证

| 编号 | 规则 | 必须 | 禁止 |
|------|------|------|------|
| R11 | 自动发送 | AI 核对所有字段无误后**直接点击发送按钮** | 调用 `ask_user_for_browser` 让用户确认或代点发送 |
| R12 | 无主题提示 | 弹出"邮件没有主题"时点击**继续发送** | 点击取消 |
| R13 | 发送后跳转 | 跳转到收件箱是**发送成功的正常行为** | 误判为发送失败而重新写信 |
| R14 | 发送验证 | 到「已发送」文件夹确认邮件存在 | 仅凭页面跳转就判定结果 |

## ⚠️ 常见错误

| ❌ 错误做法 | ✅ 正确做法 | 规则 |
|------------|-----------|------|
| 用 `evaluate` (JS) 设置输入框的 value | 用 `input` action 模拟真实键盘输入 | R5 |
| 收件人未确认标签就填主题/正文 | 先完成 Step 2a（收件人+Enter+等1s），截图确认标签后再 Step 2b 填主题+正文 | R6 |
| 收件人填中文姓名 | 填写完整邮箱地址 | R7 |
| 收件人显示 `1720778106` 就反复清除重输 | 这是自动补全的正常标签，直接继续 | R9 |
| 主题填写后用 Tab/Enter 确认 | 填完主题后直接点击正文区域 | R10 |
| 主题用 `input` action 填写 | 主题是 shadow DOM 元素，**优先用 `send_keys`** 直接打字（先 click 获取焦点） | R10 |
| 主题 input 使用 `clear: True` | 如果用 `input`，必须 `clear: False`（shadow DOM 字段 clear 会触发重置） | R10 |
| 调用 ask_user 让用户确认/代点发送 | AI 自行核对后直接点发送按钮 | R11 |
| 发送后跳转收件箱，以为失败就重写 | 去「已发送」文件夹验证邮件存在 | R13 |

## 📋 操作步骤

### Step 1 — 导航登录
1. 打开 `https://mail.qq.com`（R1）
2. 如出现登录页面且二维码右侧显示有头像列表，**直接点击第一个头像**（快捷登录），然后调用 `ask_user_for_browser` 提示用户在手机上确认登录即可（R2）
3. 如无头像快捷登录选项 → `ask_user_for_browser` 请求用户扫码/输密码（R2）
4. 等待 2-3 秒页面渲染完成（R4）

### Step 2a — 填写收件人（独立一步，等待自动补全）
1. 点击**写信**按钮进入编辑页面
2. 点击收件人输入框，用 `input` action 输入邮箱地址（R5、R7）
3. 按 **Enter** 确认 → **等待 1 秒**（R8）
4. **本步到此结束**，等截图确认收件人标签已生成（R9：数字标签是正常行为）

### Step 2b — 填写主题和正文（确认收件人后一步完成）
1. 确认截图中收件人标签已生成
2. **点击**主题字段 → **优先用 `send_keys` 直接打字**输入标题（shadow DOM 元素，`input` action 常失败；如用 `input` 则必须 `clear: False`，R10）
3. **等待 1 秒**，让 shadow DOM 渲染完成（R10）
4. **点击正文区域**（禁止用 Tab/Enter 确认主题，R10）→ 输入邮件内容
5. 如需附件：点击“添加附件”按钮上传，等待完成
6. **截图核对**所有字段是否正确

### Step 3 — 发送
1. AI 自行核对收件人、主题、正文、附件（R11）
2. 直接点击**发送按钮**（R11）
3. 如提示“无主题” → 点击**继续发送**（R12）

### Step 4 — 验证
1. 发送后自动跳转收件箱，**这是正常行为**（R13）
2. 点击左侧「已发送」文件夹（R14）
3. 确认列表中有刚才的邮件（核对收件人和主题）
4. 向用户报告发送结果

### Step 5 — 查看邮件（如需）
1. 登录后默认进入收件箱
2. 点击邮件标题查看详情
3. 提取关键信息：发件人、主题、日期、正文
