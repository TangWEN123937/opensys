## ⛔ 强制规则

### 登录与导航

| 编号 | 规则 | 必须 | 禁止 |
|------|------|------|------|
| R1 | 导航地址 | 打开 `https://www.doubao.com/chat/` | 使用带广告追踪参数的长链接 |
| R2 | 登录检测 | 检查页面是否有「登录」按钮，有则调用 `ask_user_for_browser` 请求用户登录 | 自动登录、编造账号密码 |
| R3 | 页面等待 | 每次页面操作后等待 2-3 秒 | 页面未渲染完就操作 |

### 参数设置（核心，下拉菜单是动态渲染的）

| 编号 | 规则 | 必须 | 禁止 |
|------|------|------|------|
| R4 | 下拉菜单交互 | **先点击**工具栏的「比例」或「风格」按钮 → **等待 1-2 秒**下拉菜单弹出渲染 → 再从菜单中点击选项 | 用 `evaluate`(JS) 直接设置值；不点击按钮就尝试选择选项 |
| R5 | 选项定位 | 通过下拉菜单中的**文本内容**定位选项（如 `text/4:3 文章配图，插画` 或 `text/油画`） | 用 CSS 选择器硬编码序号（菜单项顺序可能变化） |
| R6 | 菜单关闭 | 选择完一个参数后，等待 1 秒让菜单关闭，再操作下一个参数 | 连续快速操作多个下拉菜单 |
| R7 | 参数可选 | 只在 phase.details 明确指定时才修改比例/风格，未指定则保持豆包默认值 | 擅自选择比例或风格 |

### 输入与发送（核心，防止内容丢失）

| 编号 | 规则 | 必须 | 禁止 |
|------|------|------|------|
| R8 | 提示词输入 | 点击输入框获取焦点 → 优先用 **`send_keys`** 直接打字输入提示词（豆包输入框是富文本框架，`input` action 可能导致内容丢失）→ 输入完成后 **等待 2 秒**（让框架完成内部状态同步） | 用 `evaluate`(JS) 修改输入框值；用 `input` action + `clear: True`（会触发框架重置） |
| R9 | 发送前等待 | 提示词输入完成后 **必须等待 2 秒** 再点击发送按钮（`#flow-end-msg-send`） | 输入完成后立即点击发送（框架尚未同步，**内容会被清空**！）；按 Enter 发送（图像生成模式下 Enter 可能换行） |

### 等待与保存

| 编号 | 规则 | 必须 | 禁止 |
|------|------|------|------|
| R10 | 等待生成 | 发送后**每隔 5 秒截图检测**，直到看到生成的图片（通常 10-30 秒）| 发送后立刻点击（图片尚未生成） |
| R11 | 滚动到底部 | 选图前**必须先将对话区域滚动到最底部**，确保看到的是**最新一条消息**的生成结果 | 不滚动就直接点击图片（多轮对话中会误选旧消息的图片！） |
| R12 | 图片选择（AI 自主判断）| 观察最新消息中生成的 **4 张图片**，根据提示词需求**自主判断**选择最合适的一张点击进入预览（考虑构图、色彩、内容匹配度等） | 不看内容固定选第一张；选择非最新消息的旧图片 |
| R13 | 保存图片（只点一次！）| 在图片预览/详情页面点击「保存」按钮（`aria/保存`）**只点击一次** → 等待 5 秒 → 即可认为下载完成 | **反复点击保存按钮**（点击后保存按钮仍然显示在页面上是**正常行为**，不代表下载失败！每次点击都会触发一次下载，反复点击会重复下载同一张图片） |
| R14 | 下载完成判定 | 点击保存 1 次 + 等待 5 秒 = 下载完成，**直接进入下一步** | 以「保存按钮仍然存在」为由判定下载失败而反复重试 |
| R15 | 多轮生成（关闭预览）| 保存完成后，点击预览页面**右上角的关闭按钮**（`aria/保存` 旁边的 `✕` / 关闭图标）退回到对话输入页面，然后继续下一轮的参数设置和提示词输入 | 保存后不关闭预览就尝试输入新提示词；在预览页面直接操作输入框 |
| R16 | 图片归档（多轮必须！）| 每轮保存图片后、关闭预览前，**必须调用 `save_generated_image` 工具**将图片从下载目录移走并重命名（传入对应的 image_id）。⚠️ 不归档就进入下一轮，新图片会**覆盖**上一张！ | 全部生成完再统一处理；跳过归档直接开始下一轮 |

| 用 `input` action + `clear: True` 输入提示词 | 用 `send_keys` 直接打字（富文本框架 clear 会触发重置） | R8 |
| 输入提示词后立即点击发送 | 输入完成后**等待 2 秒**再点发送（框架需要同步，否则内容清空） | R9 |
| 图片还在生成中就点击 | 每 5 秒截图检测，看到完整图片后再操作 | R10 |

| 多轮生成时不归档就直接开始下一轮（下载文件被覆盖） | 每轮保存后**立即调用 `save_generated_image`** 归档，确认成功后再关闭预览 | R16 |

## 📋 操作步骤

### Step 1 — 导航登录
1. 打开 `https://www.doubao.com/chat/`（R1）
2. 等待 3 秒页面加载（R3）
3. 检查页面是否有「登录」按钮（`text/登录`）
   - 有 → 调用 `ask_user_for_browser` 提示用户在 noVNC 中完成登录，等待用户确认后继续（R2）
   - 无 → 已登录，继续下一步

### Step 2 — 新建对话 & 切换图像生成模式
1. 点击「新对话」按钮（`text/新对话` 或侧边栏顶部区域）（R3）
2. 等待 2 秒
3. 点击输入框下方工具栏的「图像生成」按钮（`text/图像生成`）
4. 等待 1 秒确认模式切换成功（输入框工具栏应出现「比例」「风格」等按钮）

### Step 3 — 设置参数（仅在 phase.details 指定时执行）

**设置比例**（如 phase.details 指定了比例）：
1. 点击工具栏中的「比例」按钮（`text/比例`）
2. **等待 2 秒**让下拉菜单弹出渲染（R4）
3. 在弹出的下拉菜单中，点击包含目标比例文本的选项（R5），例如：
   - `1:1` → 点击包含 `1:1 正方形，头像` 的菜单项
   - `4:3` → 点击包含 `4:3 文章配图，插画` 的菜单项
   - `16:9` → 点击包含 `16:9 桌面壁纸，风景` 的菜单项
4. 等待 1 秒让菜单关闭（R6）

**设置风格**（如 phase.details 指定了风格）：
1. 点击工具栏中的「风格」按钮（`text/风格`）
2. **等待 2 秒**让下拉菜单弹出渲染（R4）
3. 在弹出的下拉菜单中，点击包含目标风格文本的选项（R5），例如：
   - 油画 → 点击包含 `油画` 的菜单项
   - 动漫 → 点击包含 `动漫` 的菜单项
4. 等待 1 秒让菜单关闭（R6）

### Step 4 — 输入提示词并发送
1. 点击输入框（`aria/[role="textbox"]`）获取焦点
2. 用 **`send_keys`** 直接打字输入提示词（R8，禁止用 `input` + `clear: True`）
3. **等待 2 秒**，让富文本框架完成内部状态同步（R9，⚠️ 不等待会导致发送时内容被清空！）
4. 点击发送按钮（`#flow-end-msg-send` 区域，R9）

### Step 5 — 等待生成 & 选图保存
1. 发送后**每隔 5 秒截图检测**，等待图片生成完成（R10）
   - 通常需要 10-30 秒，最多等待 60 秒
   - 判断标准：页面出现 `<picture>` / `<img>` 元素中包含生成的图片
2. 图片生成后，**先将对话区域滚动到最底部**（R11，⚠️ 多轮对话中页面上会有多组消息，必须确保看到的是最新一条消息的结果！）
3. 观察最新消息中的 **4 张生成图片**，根据提示词需求**自主判断**选择最合适的一张，点击进入预览（R12）
4. 在预览页面点击「保存」按钮（`aria/保存`）**只点击 1 次**（R13）
5. 等待 5 秒（R14）
6. **下载完成，不要再点击保存按钮**（R13、R14）
7. **立即调用 `save_generated_image` 工具归档图片**（R16）：
   - `image_id`: 当前图片的标识（如 `img_1`，对应 image_requirements.json 中的 id）
   - `source_filename`: 留空即可（工具会自动检测最新下载的图片）
   - ⚠️ **必须在关闭预览、开始下一轮之前调用！否则下一张图片会覆盖当前图片！**

### Step 6 — 多轮生成（如需生成多张图片，对每张重复此步骤）
1. 确认 Step 5 的 `save_generated_image` 已成功返回（R16）
2. 点击预览页面**右上角的关闭按钮**（保存按钮旁边的 `✕` 关闭图标），退回到对话输入页面（R15）
3. 等待 1 秒确认回到输入页面
4. 如果下一轮需要不同的比例/风格 → 回到 **Step 3** 重新设置参数
5. 如果参数不变 → 直接回到 **Step 4** 输入新的提示词并发送
6. 发送后回到 **Step 5** 等待生成、选图保存、归档
7. 所有图片生成完成后，向用户报告生成结果

## 🔧 phase.details 格式示例

### 模式一：直接指定提示词

**单张图片**：
```
提示词：一棵苹果树，阳光下果实累累
比例：4:3
风格：油画
```

**多张图片（多轮生成）**：
```
图片1：
  提示词：一棵苹果树，阳光下果实累累
  比例：4:3
  风格：油画
图片2：
  提示词：秋天的麦田，金黄色的丰收场景
  比例：16:9
  风格：风景
图片3：
  提示词：科技感十足的AI芯片特写
  比例：1:1
  风格：赛博朋克
```

- **提示词**（必须）：生图的文字描述
- **比例**（可选）：从上表 6 种中选一个，不指定则保持豆包默认
- **风格**（可选）：从上表 30 种中选一个，不指定则保持豆包默认
- 多张图片时，按「图片N」分组，每组独立设置参数，按顺序逐轮生成

### 模式二：从配图需求文件读取（内容创作配图场景）

当 phase.details 提到 `image_requirements.json` 文件时，执行流程如下：

1. **先用 `read_local_file` 工具**读取 `drafts/image_requirements.json` 文件内容（路径在 details 中给出）
2. 解析 JSON 数组，获取每条配图需求的 `prompt`、`ratio`、`style` 字段
3. **按数组顺序逐条执行**多轮生成流程（Step 3 → Step 4 → Step 5 → Step 6 循环）
4. 每轮使用对应条目的 prompt 作为提示词，ratio 作为比例，style 作为风格（`null` 表示保持默认）

**JSON 文件格式示例**：
```json
[
  {"id": "img_1", "prompt": "科技感AI芯片微距特写，蓝色荧光电路", "ratio": "16:9", "style": "赛博朋克"},
  {"id": "img_2", "prompt": "现代化数据中心全景俯瞰图", "ratio": "16:9", "style": null}
]
```

**details 示例**：
```
读取前序写作阶段产出的 drafts/image_requirements.json 文件，按文件中的每条配图需求逐一生成图片（提示词、比例、风格均在文件中定义）
```
