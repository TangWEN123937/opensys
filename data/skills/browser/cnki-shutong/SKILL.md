## ⛔ 强制规则

### 登录阶段（先尝试自动 → 失败立即托底）

| 编号 | 规则 | 必须 | 禁止 |
|------|------|------|------|
| R1 | 入口网址 | 使用 `http://lib.shutong2.com/` | 直接访问知网公网域名 |
| R2 | 账号密码（先尝试）| **尝试**填写硬编码值：用户名 `08061539`，密码 `381005` | 不尝试直接叫用户；或者尝试多次 |
| R3 | 表单填写方式 | 用 `input` action 模拟键盘逐键输入到 `#user_name` / `#password` / `#code` | 用 `evaluate`/JS 直接赋值 `element.value`（框架不感知，内容会丢失）|
| R4 | 图形验证码（只试 1 次）| 截图后从图片中**视觉识别** `#code` 右侧的 4 位数字并填入，**只试一次** | 识别失败重试多次；刷新验证码反复试 |
| R5 | 登录点击与等待 | 点击 `#ok`（立即登录）后 **等待 5 秒** 再观察页面 | 点完立即操作 |
| R6 | 登录结果判定 | URL 跳转到 `3.shutong2.com/.../doaction.php` 视为成功 | 未跳转就继续 |
| R7 | **失败托底（核心）** | 自动登录失败（无论是验证码错/账密错/网络异常/任何原因）→ **立即**调用 `ask_user_for_browser`，提示用户：`自动登录失败，请在 noVNC（http://localhost:6080/vnc.html）中手动完成登录，完成后回复「完成」` | 尝试第 2/3/N 次自动登录；或放弃任务 |
| R8 | 入口选择（每入口每号每天 25 篇限额） | 登录成功后优先点击「知网入口2(推荐)」。若下载过程中出现无法下载（配额用尽），立即切换到下一个入口。可用入口优先级：① 知网入口2(推荐) → ② 知网入口3(推荐) → ③ 知网入口备用3(推荐) | 配额用尽后停止下载；使用非推荐入口 |
| R8a | **入口切换流程** | 当前入口无法下载时：① 关闭知网 tab → ② 切回书同入口选择页（`3.shutong2.com`）→ ③ 点击下一个入口 → ④ 重新搜索当前关键词继续下载剩余文献。3 个入口全部用尽后才放弃 | 在配额用尽的入口上反复重试 |

### 搜索阶段

|------|------|------|------|
| R13 | 下载触发 | 点击目标行的下载图标：`#gridTable tbody tr:nth-of-type(N) a.downloadlink > i`（N 从 1 开始） | 点击文献标题（会进入预览页而非下载） |
| R13a | **无下载图标则跳过** | 部分条目（如外文期刊）操作栏无下载图标（`a.downloadlink` 不存在），**直接跳过该行**，继续检查下一行，直到凑够用户要求的下载篇数 | 对无图标的条目强行点击其他按钮期待下载；因为某行没图标就停止整个下载任务 |

| R18 | 输出精简 | 每步 `evaluation` / `memory` / `next_goal` **控制在 80 字以内** | 长篇叙述看到的所有元素 |

## 📋 操作步骤

### Step 1 — 打开代理入口
1. 导航到 `http://lib.shutong2.com/`
2. 等待登录表单加载（`#user_name` / `#password` / `#code` / `#ok` 可见）

### Step 2 — 尝试自动登录（只试一次）

1. 填账号密码：
   - 点击 `#user_name` 聚焦 → `input` action 输入 `77137348`
   - 点击 `#password` 聚焦 → `input` action 输入 `217063`
2. 视觉识别验证码：
   - 截图当前页面（验证码图片在 `#code` 右侧）
   - 从截图中识别 4 位数字
   - 点击 `#code` 聚焦 → `input` action 输入识别结果
   - 不论是否有把握，**只填写一次**，不刷新验证码不重试
3. 点击 `#ok`（立即登录）
4. **等待 5 秒**后观察 URL

### Step 3 — 登录结果判定

- **成功（URL 为 `3.shutong2.com/.../doaction.php`）** → 进入 Step 4
- **失败（仍在 `lib.shutong2.com`或其他异常）** → **不重试**，立即调用 `ask_user_for_browser`：
  ```
  提示文本：自动登录失败（验证码识别错误或账密异常）。
  请在 noVNC（http://localhost:6080/vnc.html）中手动完成登录，登录成功后回复「完成」。
  ```
- 用户回复后，再次观察 URL 是否在 `3.shutong2.com`；确认后进入 Step 4

### Step 4 — 进入知网
1. 优先点击「知网入口2(推荐)」：`aria/知网入口2(推荐)` 或 `#maincolumn > div:nth-of-type(3) span:nth-of-type(2) > a`
2. 浏览器会自动切换到新 tab（`http://124.221.61.170:3344/kns8/defaultresult/index`）
3. 等待 3 秒让 SPA 完全加载，确认 `#txt_search` 可见

**入口切换（配额用尽时）**：每个入口每号每天只能下载 25 篇。若下载时出现报错/无法下载且清缓存重试后仍失败：
1. 关闭知网 tab
2. 切回书同入口选择页（`3.shutong2.com/.../doaction.php`）
3. 点击下一个入口：「知网入口3(推荐)」 或 「知网入口备用3(推荐)」
4. 重新搜索当前关键词，继续下载剩余文献

### Step 5 — 搜索文献

**若任务包含多组关键词，必须逐组搜索 + 下载，不要合并搜索（R10a）。每轮最多下载 10 篇（R10b）。**

单组关键词搜索流程：
1. 点击 `#txt_search` 聚焦
2. 用 `input` action **一次性填入**当前这组关键词（英文/中文均可）
3. 按 **Enter** 触发搜索
4. 等待结果列表 `#gridTable` 加载 2-3 秒
5. 进入 Step 6 → Step 7/8 下载（本轮最多 10 篇）
6. 本轮下载完成后，若还有下一组关键词：清空搜索框（全选 + Delete）→ 回到第 1 步输入下一组关键词

### Step 6 — 确认相关度排序（默认已生效，无需操作）
1. 知网搜索结果默认按相关度排序，**不需要点击任何排序按钮**
2. 可选：截图确认排序栏中「相关度」是否高亮
3. 若用户明确要求其他排序方式，用 `evaluate` action 执行 JS：
   - 时间 → `document.querySelector('#PT a').click()`
   - 被引 → `document.querySelector('#CF a').click()`
   - 下载量 → `document.querySelector('#DFR a').click()`
   - 相关度 → `document.querySelector('#FFD a').click()`（默认已选中）
4. 切换排序后 **等待 3 秒** 让列表刷新，截图确认生效。**若 2 次 JS click 后仍未生效 → 放弃，按当前顺序下载**

**⚠️ 重要区分**：排序按钮（`#FFD`/`#DFR`/`#PT`/`#CF` 内的 `<a>`）在搜索结果列表**上方**的排序栏中；每条结果行内的下载图标（`a.downloadlink`）在行**右侧**操作栏中。两者完全不同，绝对不要混淆。

### Step 7 — 单篇下载
_（如果只要一篇，走本步；多篇走 Step 8）_

1. 点击 `#gridTable tbody tr:nth-of-type(1) a.downloadlink > i`
2. 会打开新 tab，**等待 5 秒**让浏览器完成下载
3. 关闭下载 tab，切回搜索结果 tab
4. 从 `/app/data/downloads/` 读取新下载的文件路径，作为产出物

### Step 8 — 批量下载（严格循环流程，本站无批量功能）

**假设本轮要下载 N 篇（由用户指定，如 N=5，单轮上限 10 篇）**：

注意：并非每行都有下载图标，需按行遍历、跳过无图标行、直到成功下载 N 篇。若需更多文献，换下一组关键词重新搜索（R10a）。

```
row = 1, downloaded = 0
当 downloaded < N 时循环:
  ⓪ 检查第 row 行是否存在下载图标 `#gridTable tbody tr:nth-of-type(row) a.downloadlink`
     - 若不存在 → row++，跳过该行继续循环（R13a）
     - 若已无更多行（翻到列表末尾）→ 退出循环
  ① 在搜索结果 tab 上，点击 `#gridTable tbody tr:nth-of-type(row) a.downloadlink > i`
  ② 浏览器自动打开新 tab（下载详情页或直接触发下载）
  ③ 等待 5 秒（让浏览器完成下载到 /app/data/downloads/）
  ③-ERR 若新 tab 显示下载错误（错误处理流程 R16）:
      a. 关闭报错 tab
      b. 切回搜索结果 tab
      c. 点击页面上「下载错误？点此清除缓存」按钮
      d. 等待弹出的新 tab 加载完成后关闭该 tab
      e. 切回搜索结果 tab
      f. 重试点击第 i 篇的下载图标（回到 ① 重试一次）
      g. 若重试仍报错 → 跳过该篇，row++，继续循环
  ④ 关闭当前新 tab（browser-use 的 close_tab 动作）
  ⑤ 切回搜索结果 tab（应为 index=0 或标题含 "知网" 的 tab）
  ⑥ 确认搜索结果列表仍正常显示（默认相关度排序，无需额外操作）
  ⑦ downloaded++, row++，继续循环
```

**关键点**：
- 每篇下载都要完整走完 ①→⑦，**禁止** 不关 tab 就点下一篇
- 遇到下载错误必须走 ③-ERR 清缓存流程（关报错 tab → 点清缓存 → 关缓存 tab → 回搜索页重试），每篇最多重试 1 次
- 每次循环完都要 **截图确认**搜索结果页可见且排序正确
- 循环结束后，统计 `/app/data/downloads/` 中的新文件，汇报给用户

### Step 9 — 完成报告（最后一步）
1. 列出已下载文件路径 + 每篇标题（从搜索列表提取）
2. 若某篇下载失败，说明第几篇 + 失败原因（如 tab 未关成功 / 列表重置未恢复 / 文件未落盘）

## 🔑 已知坑点

- **登录策略**：硬编码账密 + AI 视觉识别验证码 先试一次，失败立即 `ask_user_for_browser` 托底（用户在 noVNC 手动登录）。不重试多次是为了避免卡死流程
- **验证码 AI 识别**：Qwen / Claude / Gemini 等视觉模型都能识别清晰的 4 位数字验证码，识别成功率约 60-80%；一次不中就移交用户，比反复刷验证码更快
- **input action vs evaluate**：知网登录页和搜索框都是标准 form，**必须用 input action 模拟键盘**；用 evaluate 赋值 `element.value` 虽然能改显示值，但框架（Vue/Angular/原生 form）的 state 不会更新，提交时字段为空
- **批量下载 tab 累积**：如果不关闭下载 tab，10 篇后浏览器会有 10 个僵尸 tab，拖慢响应甚至内存溢出
- **排序重置风险**：极少数情况下关闭子 tab 切回后排序可能重置；由于默认就是相关度排序，重置后也不影响结果
- **下载目录**：`/app/data/downloads/`（已在 config 中配置）
- **排序 DOM 结构**：排序栏是 `<ul id="sortList">`，每个排序项是 `<li id="FFD/DFR/PT/CF">`，内含 `<a>` 子元素。**需要切换排序时必须用 JS `document.querySelector('#XX a').click()` 点击 `<a>` 才能触发排序**。直接用 click action 点击 `<li>` 无效。排序按钮和行内下载图标（`a.downloadlink`）完全无关
- **排序字段 id 对照**：
  - `#FFD` = 相关度 ← **默认（搜索后自动生效，`class="DESC cur"` 高亮，无需操作）**
  - `#DFR` = 下载量
  - `#PT` = 发表时间
  - `#CF` = 被引频次
- **中文输入法 IME**：若用 keyDown 逐字输入会触发 Process 键（见 R10）
- **下载报错恢复**：知网偶发下载错误，新 tab 出现报错页面时走 R16 清缓存流程（关报错 tab → 点清缓存 → 关缓存 tab → 重试），每篇最多重试 1 次
- **LLM 输出防护**：`max_completion_tokens=16384` + `json_repair` 兜底；仍建议 evaluation/memory 控在 80 字内（R18）
