## 内容撰写流程（本技能被激活时执行）

### 第一步：确认写作参数

在开始撰写前，确保已明确以下信息（如不清楚，使用 ask_user 询问）：
1. **目标平台**：公众号、博客、技术文档、社交媒体等
2. **目标受众**：技术人员、普通读者、决策者等
3. **风格要求**：专业严谨、轻松活泼、简洁直白等
4. **字数范围**：大致字数目标
5. **大纲/结构**：如果已有大纲，严格按照大纲执行

### 第二步：逐节撰写

1. 按大纲顺序逐节撰写，每节完成后检查与上一节的衔接
2. 开头段要有吸引力，点明核心价值或痛点
3. 每段围绕一个核心观点，段内逻辑清晰
4. 使用过渡句连接各节，保持整体流畅
5. 结尾段总结要点，给出行动建议或展望

### 第三步：配图需求标注（当任务要求配图时执行）

如果用户需求或大纲中提及配图、插图、图文并茂等要求，在撰写过程中**同步标注配图需求**：

1. 撰写每个段落/章节时，判断该位置是否需要配图
2. 需要配图时，根据**段落上下文内容**撰写具体的图像生成提示词（描述画面内容、风格、色调等）
3. 所有配图需求汇总后，用 `write_and_run_script` 保存为 JSON 文件

**配图需求文件格式**（保存到 `<任务目录>/drafts/image_requirements.json`）：

```json
[
  {
    "id": "img_1",
    "position": "chapter_1 段落2后",
    "description": "用于说明AI芯片的微观结构",
    "prompt": "科技感十足的AI芯片微距特写，电路纹路发出蓝色荧光，背景为暗色调",
    "ratio": "16:9",
    "style": "赛博朋克"
  },
  {
    "id": "img_2",
    "position": "chapter_2 开头",
    "description": "展示数据中心规模",
    "prompt": "现代化数据中心全景俯瞰图，整齐排列的服务器机架，冷蓝色灯光",
    "ratio": "16:9",
    "style": null
  }
]
```

**字段说明**：
- `id`：图片唯一标识（img_1, img_2, ...）
- `position`：图片在文章中的插入位置（对应 drafts/ 中的章节文件名 + 段落位置）
- `description`：简要说明为什么需要这张配图（给后续排版阶段参考）
- `prompt`：**图像生成提示词**（⭐ 核心字段，要根据段落内容写出具体、生动的画面描述）
- `ratio`：图片比例，可选值：`1:1` / `2:3` / `3:4` / `4:3` / `9:16` / `16:9`，不指定填 `null`
- `style`：风格，如 `油画` / `赛博朋克` / `动漫` 等，不指定填 `null`

**提示词撰写要点**：
- 基于段落实际内容描述画面，不要写空泛的描述
- 包含主体、场景、色调、氛围等具体元素
- 如果是技术类文章，优先使用写实/科技风格
- 如果是生活类文章，可考虑插画/水彩等风格

> **注意**：如果任务未要求配图，则跳过此步骤，不生成 image_requirements.json。

### 第四步：自检清单

撰写完成后，逐条自检：
- [ ] 是否覆盖了大纲中的所有要点
- [ ] 段落之间是否有逻辑断层
- [ ] 是否有重复或冗余的内容
- [ ] 语言风格是否统一
- [ ] 字数是否在目标范围内
- [ ] 专业术语是否准确、是否需要解释
- [ ] 如需配图：是否已生成 `image_requirements.json`，提示词是否具体且与段落内容匹配

### 第五步：保存文件（必须）

**每个子任务完成撰写后，必须使用 `write_and_run_script` 工具将内容写入文件：**

```python
# 示例：使用 write_and_run_script 保存撰写内容（草稿写入 drafts/）
content = """# 文章标题

## 第一节

正文内容...
"""

import os
drafts_dir = "<任务目录>/drafts"  # 如 /app/output/20260420_1435_xxx/drafts
os.makedirs(drafts_dir, exist_ok=True)
with open(f"{drafts_dir}/chapter_1.md", "w", encoding="utf-8") as f:
    f.write(content)
print(f"已保存到 drafts/，共 {len(content)} 字符")
```

- 撰写阶段的草稿统一写入 `<任务目录>/drafts/`（如 `drafts/chapter_1.md`、`drafts/section_2.md`）
- **不要直接写入 `output/`** — `output/` 仅用于格式化阶段输出的最终交付物
- 每个子任务保存自己负责的部分
- 禁止仅在消息中输出长文本而不保存文件

### 注意事项

- **草稿写入 drafts/**：每个子任务必须用 `write_and_run_script` 将撰写内容保存为 .md 文件到 `<任务目录>/drafts/` 目录，终稿由后续格式化阶段整合
- **并行撰写时**：确保各节的术语、风格一致
- **必须检测字数**：字数必须在安排给的字数范围内，否则就修改，知道满足字数要求为止。
