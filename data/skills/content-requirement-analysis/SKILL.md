## 内容需求分析流程（本技能被激活时执行）

在开始任何内容创作之前，必须先完成以下需求确认。使用 `ask_user` 工具逐项确认不明确的参数。

### 第一步：明确内容基本参数

逐项确认以下信息（如用户未提供，必须用 ask_user 询问）：

1. **目标平台**：公众号、技术博客、知乎、小红书、抖音文案、企业官网等
2. **目标受众**：
   - 年龄段和专业背景
   - 阅读场景（通勤碎片时间 / 办公学习 / 休闲浏览）
   - 对主题的了解程度（小白 / 有基础 / 专业人士）
3. **内容主题**：核心论点或要传达的关键信息
4. **风格要求**：专业严谨 / 轻松活泼 / 简洁直白 / 故事叙述
5. **字数范围**：大致字数目标（如 2000-3000 字）
6. **特殊要求**：是否需要配图说明、是否有参考文章、是否有品牌调性要求

### 第二步：分析内容方向

基于确认的参数，输出以下分析：

1. **内容定位**：一句话概括这篇内容要解决什么问题 / 传递什么价值
2. **差异化角度**：与同类内容相比，本篇的独特切入点是什么
3. **核心读者收益**：读者看完能获得什么（知识 / 方法 / 启发 / 情感共鸣）

### 第三步：输出需求确认单

用以下格式整理确认结果，供后续阶段参考：

```
## 内容需求确认单

- 平台：[目标平台]
- 受众：[目标读者画像]
- 主题：[核心主题]
- 风格：[写作风格]
- 字数：[目标字数范围]
- 特殊要求：[如有]
- 内容定位：[一句话定位]
- 差异化角度：[切入点]
- 读者收益：[核心价值]
```

### 注意事项

- **不要假设参数**：宁可多问一个问题，也不要自行猜测用户意图
- **不要跳过确认**：即使用户描述很详细，也要回顾确认是否有遗漏
- **简洁提问**：一次 ask_user 可以包含多个问题，避免反复打断用户
