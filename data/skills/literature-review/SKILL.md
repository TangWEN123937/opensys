## 文献综述流程（本技能被激活时执行）

### 前置条件
- Phase 3（PDF 向量化）已完成，文献已入库 ChromaDB `documents` collection
- Phase 1 需求确认单中有明确的研究主题和检索关键词

### 向量知识库与检索脚本说明

#### 数据库信息
- **数据库**：ChromaDB（本地持久化），路径 `/app/data/chroma_db`
- **集合名称**：`documents`（所有 PDF 文献统一存储）
- **向量模型**：BGE-M3 Embedding（本地服务 `http://host.docker.internal:8100/api/v1/embed`）
- **Metadata 字段**：`source_file`（来源 PDF 文件名）、`section`（章节标题）、`topic`（LLM 自动分类的主题）、`doc_id`（PDF SHA256 前缀）

#### 预置检索脚本
- **路径**：`/app/data/skills/paper-writing/query_references.py`
- **检索方式**：混合检索（语义相似度 × 0.7 + 关键词匹配 × 0.3），Embedding 服务不可用时自动降级为纯关键词匹配
- **输出格式**：结构化 Markdown，每条结果包含来源文件、章节、综合/语义/关键词得分、文献正文片段（超过 1500 字自动截断）
- **参数**：
  - `--query`（必填）：检索关键词，支持自然语言描述
  - `--topic`（可选）：按 metadata.topic 过滤，限定主题分类
  - `--top_k`（可选）：返回条数，默认 10

### 第一步：知识库检索与文献梳理

使用 `run_terminal` 调用预置脚本检索已入库文献（**禁止自行编写 ChromaDB 查询脚本**）：

```bash
# 按主题语义检索（每个研究问题分别检索一次）
python /app/data/skills/paper-writing/query_references.py --query "城市排水管网改造技术" --top_k 10
python /app/data/skills/paper-writing/query_references.py --query "非开挖修复 CIPP 技术" --topic "排水管网" --top_k 8
python /app/data/skills/paper-writing/query_references.py --query "城市更新背景 管网运维" --top_k 8
```

对检索结果逐篇整理：
1. **文献基本信息**：作者、标题、来源文件
2. **核心观点**：该文献的主要论点和结论
3. **研究方法**：采用什么研究方法（定性/定量/混合）
4. **关键数据**：重要的统计数据或实验结果
5. **与本研究的关系**：支持/反对/补充本研究的哪个方面

### 第二步：网络补充调研

使用 `web_tool` 补充知识库中可能缺失的内容：

1. **最新研究动态**：搜索近 1-2 年的最新研究成果
2. **权威综述文章**：搜索该领域已有的综述论文，了解整体研究脉络
3. **政策/行业报告**：如果涉及应用领域，补充相关政策文件或行业数据
4. **国际研究对比**：补充国外相关研究（如适用）

### 第三步：文献综述撰写

按以下结构组织文献综述：

```markdown
## 文献综述

### X.1 研究脉络与发展历程
- 该领域的研究起源和发展阶段
- 关键里程碑事件或标志性研究
- 研究重心的演变趋势

### X.2 主要研究流派与观点
- 流派 A：代表学者 + 核心观点 + 主要论据（引用标记 [1][2]）
- 流派 B：代表学者 + 核心观点 + 主要论据（引用标记 [3][4]）
- 各流派之间的争论焦点和共识

### X.3 研究方法综述
- 该领域常用的研究方法
- 各方法的优缺点和适用场景
- 方法论上的创新趋势

### X.4 关键发现与研究成果
- 已有研究的主要发现（按主题分组）
- 重要的数据和结论
- 研究成果的实际应用情况

### X.5 研究空白与不足
- 现有研究的局限性
- 尚未解决的问题
- 本研究的切入点和价值所在

### X.6 本研究定位
- 本研究与现有文献的关系
- 本研究要填补的具体空白
- 本研究的预期贡献
```

### 第四步：保存文件（必须）

**每个子任务完成撰写后，必须使用 `write_and_run_script` 工具将内容写入文件：**

```python
# 示例：使用 write_and_run_script 保存文献综述
content = """## 文献综述

### 2.1 研究脉络与发展历程

...
"""

import os
output_dir = "<工单中的输出目录>"  # 如 /app/output/20260420_1435_xxx/output
os.makedirs(output_dir, exist_ok=True)
with open(f"{output_dir}/literature_review.md", "w", encoding="utf-8") as f:
    f.write(content)
print(f"已保存文献综述，共 {len(content)} 字符")
```

- 文件路径格式：`<输出目录>/literature_review.md` 或按主题分文件
- 文献引用清单单独保存为 `<输出目录>/references.md`

### 产出物

- 结构化文献综述初稿（写入输出目录的 .md 文件）
- 文献引用清单（作者-年份-标题-来源，供参考文献阶段使用）

### 注意事项

- **标注来源**：每个论点必须标注来源文献（使用引用标记如 [1][2]），禁止无来源的断言
- **客观中立**：文献综述应客观呈现各方观点，不带个人偏见
- **逻辑递进**：从宏观到微观、从历史到现状、从共识到分歧
- **突出空白**：文献综述的核心目标是论证"本研究的必要性"，必须清晰指出研究空白
- **避免堆砌**：不要简单罗列文献，要有综合分析和批判性思考
- **引用数量**：综述部分引用文献数量应不少于总参考文献的 60%
- **禁止** 自行编写 ChromaDB 查询脚本或直接操作 ChromaDB，必须使用预置的 `query_references.py`
- **禁止** 使用 `write_and_run_script` 编写数据库查询代码替代 `query_references.py`
- **必须输出文件**：每个子任务必须用 `write_and_run_script` 将综述内容保存为 .md 文件到「工单」指定的输出目录，禁止仅在消息中输出长文本
